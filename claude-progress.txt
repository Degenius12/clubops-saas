   - Cause: Creating new rate limiter instance per request
   - Fix: Moved limiter creation to module initialization (create once, reuse)
   - Impact: All API requests were getting validation errors

   - File: backend/middleware/rateLimit.js
   - Error: `Custom keyGenerator appears to use request IP without calling ipKeyGenerator helper`
